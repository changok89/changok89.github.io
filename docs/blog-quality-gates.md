# Blog Quality Gates

AdSense 친화적인 개발 블로그 품질을 위해 아래 게이트를 사용합니다.

## Hard blockers

아래 중 하나라도 해당하면 **발행 금지**입니다.

- 본문이 지나치게 짧음
  - 권장 최소: 900+ words
  - 절대 하한: 450 words
- 제목 대비 실제 정보량이 부족함
- 외부 자료 요약만 있고 자기 관점/실전 팁이 없음
- 코드/명령어/예시/실패 사례/비교가 거의 없음
- 서론만 있고 문제 해결 단계가 없음
- 문서가 복붙처럼 보이거나 상투 표현이 과도함
- front matter 누락(제목/날짜/카테고리/태그/요약)
- 모바일/개발자 독자에게 어떤 상황에서 유용한지 설명이 없음

## Strong publish signals

아래 요소가 많을수록 좋습니다.

- 실제 에러 상황 또는 실무 배경 설명
- Android/iOS/mobile/AI/on-device AI/SDD/vibe coding 맥락 연결
- 실행 가능한 코드, 명령어, 설정 조각
- 버전/환경/전제조건 명시
- 흔한 함정과 디버깅 포인트
- 대안 비교 또는 선택 기준
- 결론에서 “언제 이 방법을 쓰면 좋은지” 정리

## Review checklist

### 1) Search intent
- 독자가 검색해서 들어왔을 때 첫 30초 안에 필요한 답을 얻는가?
- 제목과 본문이 정확히 일치하는가?

### 2) Depth
- 초보 설명만 있거나 명령어 나열로 끝나지 않는가?
- 왜 그런지, 언제 쓰는지, 어디서 깨지는지 설명하는가?

### 3) Originality
- 내 경험/판단/비교/선호가 들어가 있는가?
- “공식 문서 링크 하나면 끝나는 내용”을 그대로 반복하지 않는가?

### 4) Helpfulness
- 바로 따라할 수 있는가?
- 스크린샷, 코드 블록, 단계, 체크포인트가 충분한가?

### 5) AdSense-friendliness
- 페이지 체류 시간을 만들 만한 실질 내용이 있는가?
- 얇은 콘텐츠, 자동생성 티, 과도한 키워드 반복이 없는가?

## Default scoring heuristic

자동 점검 스크립트는 아래 항목을 기반으로 품질을 추정합니다.

- 단어 수
- 제목/요약 존재 여부
- 헤딩 수
- 코드 블록 수
- 링크 수
- 이미지 수
- 리스트 편중 여부
- 주제 적합도(Android/iOS/mobile/AI 계열 키워드)

점수는 참고용입니다. 최종 발행 결정은 사람 리뷰가 합니다.

## Practical thresholds used by this repo

- **Good**: 75+
- **Needs work**: 55-74
- **Thin / upgrade candidate**: below 55

추가 경고:

- 450 words 미만
- 헤딩 2개 미만
- 코드/예시/링크가 거의 없음
- excerpt가 너무 짧음

## Safe publishing rule

자동화는 낮은 점수 글을 **조용히 발행하지 않습니다**.

발행 전에 최소한 다음 3가지를 만족해야 합니다.

1. 품질 점수 75 이상 또는 수동 승인 메모 존재
2. 리뷰 체크리스트 확인 완료
3. 초안 상태가 `needs_review`에서 `approved`로 명시적으로 변경됨
