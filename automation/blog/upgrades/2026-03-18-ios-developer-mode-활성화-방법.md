# iOS Developer Mode 활성화 방법

iPhone을 Mac에 연결해서 바로 실행해 보려는데 Xcode가 기기를 잡지 못하거나, 앱 설치 단계에서 개발자 모드가 꺼져 있다는 경고가 뜨는 경우가 있다. 예전에는 케이블만 꽂으면 되는 느낌이었는데, iOS 16 이후에는 **Developer Mode**가 별도 보안 단계로 들어가면서 초반 진입 장벽이 조금 생겼다. 처음 설정할 때는 왜 막히는지 애매해서 시간을 꽤 쓰기 쉽다.

이 글은 “설정 메뉴가 어디 있지?” 수준에서 끝나는 글이 아니라, **언제 필요한지**, **왜 켜야 하는지**, **켜도 여전히 설치가 안 될 때 어디를 봐야 하는지**까지 정리한 실무용 메모다. iPhone, iPad, 일부 watchOS 테스트 환경에서도 비슷한 흐름으로 이해하면 된다.

## Developer Mode가 왜 필요한가

Apple 입장에서는 일반 사용자 기기에 개발용 앱, 내부 테스트 앱, 자동화 도구가 너무 쉽게 올라가는 것을 막고 싶다. 그래서 App Store나 TestFlight처럼 검증된 경로는 그대로 두되, Xcode에서 직접 올리거나 개발용 서명된 앱을 설치하는 경우에는 **사용자가 명시적으로 기기를 개발 대상으로 허용**해야 한다.

정리하면 Developer Mode는 아래 상황에서 중요하다.

- Xcode에서 디바이스에 직접 앱 설치
- 사내 배포 앱이나 디버그 빌드 테스트
- Appium 같은 자동화 테스트 도구 사용
- 로그 수집, 디버깅, 성능 측정 등 개발용 연결

반대로 App Store 설치 앱을 그냥 쓰는 일반 사용자라면 이 기능을 켤 이유가 거의 없다.

## 활성화 전에 먼저 확인할 것

Developer Mode 메뉴가 안 보이는 경우도 있다. 그럴 때는 메뉴 위치만 찾기보다 아래 조건을 먼저 확인하는 편이 빠르다.

1. **iOS 버전 확인**: 주로 iOS 16 이상에서 이 흐름이 명확하게 보인다.
2. **Mac과 신뢰 관계 확인**: 처음 연결했다면 “이 컴퓨터를 신뢰하시겠습니까?”를 승인해야 한다.
3. **Xcode 버전 확인**: 최신 iOS를 붙이는데 Xcode가 너무 오래되면 다른 문제처럼 보일 수 있다.
4. **케이블 상태 확인**: 충전은 되는데 데이터 전송이 안 되는 케이블이 은근히 많다.

특히 현장에서 자주 겪는 문제는 “설정에서 메뉴가 안 보인다”가 아니라 사실은 **기기가 아직 개발 연결을 한 번도 제대로 시도하지 않은 상태**인 경우다.

## 실제 활성화 순서

기본 흐름은 단순하다.

1. iPhone을 Mac에 USB로 연결한다.
2. Xcode에서 실행 대상에 해당 기기가 보이는지 확인한다.
3. 필요하면 한 번 빌드/실행을 시도한다.
4. iPhone에서 Developer Mode 관련 안내가 나타나면 허용한다.
5. 기기가 재시동된다.
6. 재시동 후 다시 한 번 사용 여부를 묻는데 여기서 최종 승인한다.
7. 기기 암호를 입력하면 활성화 완료다.

기기 자체 메뉴로 들어가 직접 찾으려면 보통 다음 경로다.

- **설정 > 개인정보 보호 및 보안 > 개발자 모드**

활성화 후에는 재부팅이 한 번 들어가고, 재부팅 후 재확인까지 해야 완전히 켜진다. 이 두 번째 확인 단계를 빼먹으면 분명 켠 것 같은데 Xcode에서는 계속 비활성 상태처럼 보일 수 있다.

## Xcode에서 계속 막힐 때 보는 포인트

실제로는 Developer Mode만 켠다고 끝나지 않는 경우가 많다. 아래 순서대로 보면 대체로 해결된다.

### 1) 신뢰 팝업을 놓친 경우

Mac 연결 시 뜨는 “이 컴퓨터를 신뢰” 팝업을 취소했거나 무심코 넘기면 개발자 모드보다 앞단에서 막힌다. 이 경우는 케이블을 다시 연결하고, 잠금 해제된 상태에서 다시 승인해 주는 것이 먼저다.

### 2) Xcode와 iOS 버전 미스매치

신형 iOS 베타나 최신 정식 버전을 쓰는데 Xcode가 예전 버전이면 DeviceSupport 문제 때문에 연결이 이상하게 보일 수 있다. “개발자 모드가 안 된다”로 오해하기 쉽지만 실제 원인은 툴체인 버전이다.

### 3) MDM 또는 회사 정책

회사 지급 기기에서는 보안 정책 때문에 일부 설정이 제한될 수 있다. 설정 화면은 보여도 실제 활성화가 막히거나, 내부 프로파일이 충돌하는 경우가 있다. 이때는 개인 기기 기준 가이드만 믿고 밀어붙이면 안 된다.

## 자주 겪는 실패 사례

### 메뉴가 안 보인다

가장 먼저 iOS 버전을 확인한다. 그리고 Mac에 연결한 뒤 Xcode에서 해당 기기를 한 번이라도 개발 대상으로 인식시켜 보는 것이 좋다. 그냥 설정 앱만 뒤지면 안 보이는 것처럼 느껴질 수 있다.

### 켰는데 앱 설치가 계속 실패한다

대부분 아래 셋 중 하나다.

- 재부팅 후 최종 확인을 하지 않음
- Mac 신뢰 승인 누락
- 서명/프로비저닝 문제를 Developer Mode 문제로 착각

즉, 개발자 모드는 필요조건이지 항상 충분조건은 아니다.

### 팀원이 “나는 메뉴가 없는데?”라고 한다

iPadOS, iPhone 모델, iOS 버전, 회사 정책 여부, 연결한 Mac 환경이 모두 다를 수 있다. 그래서 팀 위키에 단순히 “설정 > 개발자 모드 ON” 한 줄만 적어두면 재현이 안 된다. 실제로는 **연결 → 승인 → 재부팅 → 재승인** 흐름까지 같이 적어두는 편이 훨씬 낫다.

## 실무에서의 판단

개인적으로는 테스트 전용 기기에는 Developer Mode를 켜두고, 일상용 메인폰은 필요할 때만 켜는 편이 관리가 쉽다. 이유는 단순하다. 개발용 기기는 실험성 앱, 사내 빌드, 자동화 도구가 자주 올라가고, 메인폰은 그럴 필요가 거의 없기 때문이다.

또 팀 온보딩 문서에는 아래 두 링크를 같이 묶어 두는 편이 좋다.

- `iOS 화면 미러링`
- `iMazing 사용법`

실제 디바이스 테스트 세팅은 Developer Mode 하나로 끝나지 않고, 화면 공유나 파일 접근 같은 주변 도구까지 연결되는 경우가 많아서다.

## 빠른 체크리스트

- [ ] iPhone 잠금 해제 상태로 Mac에 연결했다
- [ ] 이 컴퓨터를 신뢰 팝업을 승인했다
- [ ] Xcode 버전이 현재 iOS와 크게 차이나지 않는다
- [ ] 설정 > 개인정보 보호 및 보안에서 개발자 모드를 확인했다
- [ ] 재시동 후 최종 승인과 암호 입력까지 완료했다
- [ ] 그래도 안 되면 서명/프로비저닝 문제를 따로 확인했다

## 마무리

iOS Developer Mode는 기능 자체는 단순하지만, 처음에는 “왜 설치가 안 되지?”라는 막연한 막힘으로 체감되는 경우가 많다. 메뉴 위치만 외우기보다 **보안 승인 절차**라고 이해하면 훨씬 덜 헷갈린다. 특히 팀 문서에는 메뉴 경로보다도 **연결, 신뢰, 재부팅, 재승인** 순서를 남겨두는 쪽이 실전에서 더 도움이 된다.
