# Blog Review Checklist

- [ ] 제목과 본문이 정확히 맞는다.
- [ ] 독자 문제를 첫 부분에서 바로 설명한다.
- [ ] 실전 단계, 코드, 명령어, 예시 중 최소 하나 이상 있다.
- [ ] 왜 이 방법이 필요한지 맥락이 있다.
- [ ] Android / iOS / mobile / AI / on-device AI / SDD / vibe coding 중 적절한 연결이 있다.
- [ ] 상투적인 문장, 반복, 불필요한 군더더기를 제거했다.
- [ ] 복붙처럼 보이는 문장이 없다.
- [ ] 얇은 글처럼 보이지 않을 정도로 충분한 깊이가 있다.
- [ ] 더 나은 대안 또는 주의점이 있다.
- [ ] 최종 상태를 `approved`로 명시하기 전까지 업로드하지 않는다.
