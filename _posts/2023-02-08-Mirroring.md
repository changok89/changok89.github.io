iPhone이나 iPad 화면을 외부 화면에 보여줘야 하는 상황은 생각보다 자주 생긴다. 예를 들면 아래와 같다.

- 앱 시연이나 회의 중 데모
- 화면 녹화 자료 제작
- 모바일 UI 리뷰
- 버그 재현 장면 공유
- 고객 또는 동료에게 설정 방법 안내

문제는 "미러링"이라고 해도 목적에 따라 적합한 방법이 다르다는 점이다. 어떤 경우는 Mac으로 화면만 크게 보여주면 되고, 어떤 경우는 무선으로 TV나 회의실 디스플레이에 띄우면 된다. 또 어떤 경우는 조작까지 가능한 서드파티 도구가 필요하다.

이 글에서는 내가 실제로 자주 쓰는 iOS 화면 미러링 방법을 용도별로 정리한다.

## 먼저 정리: 어떤 방법을 써야 하나

가장 간단한 기준은 아래처럼 보면 된다.

- **Mac에서 유선으로 안정적으로 보여주기** → QuickTime Player
- **Apple TV나 AirPlay 지원 디스플레이에 무선 전송** → AirPlay
- **추가 기능(녹화, 제어, 스트리밍 등)이 필요** → 서드파티 도구

## 1. QuickTime Player로 미러링하기

QuickTime Player는 macOS에 기본 포함되어 있어서 별도 설치 없이 가장 빠르게 쓸 수 있는 방법이다. 특히 맥북 화면과 아이폰 화면을 동시에 보여주면서 설명해야 할 때 편하다.

### 장점

- macOS 기본 앱이라 바로 사용 가능
- 별도 회원가입이나 광고가 없음
- 유선 연결이라 비교적 안정적
- 화면 녹화까지 같이 진행하기 쉬움

### 단점

- 보통 iPhone을 **Mac에서 원격 조작**할 수는 없음
- 케이블 연결이 필요함
- 발표 현장에서 무선 편의성은 AirPlay보다 떨어짐

### 사용 방법

1. iPhone 또는 iPad를 Mac에 케이블로 연결한다.
2. macOS에서 **QuickTime Player**를 실행한다.
3. 상단 메뉴에서 **파일 > 새로운 동영상 녹화**를 선택한다.
4. 녹화 버튼 옆의 아래 화살표를 클릭한다.
5. 카메라 또는 입력 소스 목록에서 연결된 iPhone/iPad를 선택한다.

![Image Alt Mirroring1](/assets/img/contents/mirroring/mirroring1.png)

선택이 정상적으로 되면 iPhone 화면이 QuickTime 창 안에 바로 표시된다.

### QuickTime 방식이 잘 맞는 상황

- 개발 중 디버깅 장면을 화면 녹화할 때
- 실기기 화면을 Mac 발표 자료와 함께 보여줘야 할 때
- 무선 지연보다 유선 안정성이 중요한 상황

### 자주 막히는 문제

#### 기기가 목록에 안 뜬다
- 케이블 또는 허브 문제일 수 있다.
- 기기 잠금이 해제되어 있어야 한다.
- "이 컴퓨터를 신뢰" 팝업을 허용해야 한다.
- Finder/iTunes 쪽에서 기기 인식이 먼저 되는지 확인한다.

#### 소리는 안 잡힌다
QuickTime의 목적이 화면 미러링인지, 오디오 입력까지 포함인지에 따라 별도 선택이 필요하다. 앱 데모만 하는 경우라면 화면만으로 충분한 경우가 많다.

## 2. AirPlay로 무선 미러링하기

AirPlay는 iPhone이나 iPad 화면 전체를 Apple TV 또는 AirPlay 지원 디스플레이로 보내는 방식이다. 케이블 없이 회의실 TV나 대형 모니터에 띄울 때 편하다.

### 장점

- 케이블 없이 무선으로 연결 가능
- 발표, 회의, 교육 상황에서 편리함
- iPhone/iPad 단독 조작 흐름이 자연스러움

### 단점

- 네트워크 상태 영향을 받음
- 지연이 있을 수 있음
- 회사/공용망에서는 연결이 제한될 수 있음

### 사용 방법

1. iPhone 또는 iPad에서 제어 센터를 연다.
2. **화면 미러링(Screen Mirroring)** 을 선택한다.
3. 목록에서 Apple TV 또는 AirPlay 지원 디스플레이를 선택한다.
4. 필요하면 화면에 표시되는 코드를 입력한다.

| ![Image Alt Mirroring2](/assets/img/contents/mirroring/mirroring2.png) | ![Image Alt Mirroring2](/assets/img/contents/mirroring/mirroring3.png) |

### AirPlay가 잘 맞는 상황

- 회의실 TV로 앱 데모를 할 때
- 집이나 사무실에서 무선으로 크게 보고 싶을 때
- Mac 없이 iPhone 단독으로 발표 흐름을 만들고 싶을 때

### 주의할 점

- 동일 네트워크 또는 AirPlay 가능한 환경이 필요하다.
- 회사 네트워크 정책상 peer discovery가 막혀 있으면 연결이 실패할 수 있다.
- 영상 재생과 조작이 동시에 많은 화면에서는 지연이 느껴질 수 있다.

## 3. 서드파티 프로그램 사용

기본 기능만으로 부족할 때는 전용 미러링 도구를 고려할 수 있다. 대표적으로 아래 같은 도구들이 있다.

- iMyFone MirrorTo
- LetsView
- Vysor

각 도구마다 강점이 다르다.

### 서드파티 도구를 쓰는 이유

- 화면 녹화/스트리밍이 더 편함
- 원격 조작이나 추가 부가기능이 있을 수 있음
- Windows, macOS를 모두 지원하는 경우가 많음
- 발표용 오버레이, 녹화 품질 옵션이 더 많음

### 단점

- 유료 기능 제한이 많을 수 있음
- 기기 권한 요청이나 드라이버 설치가 필요할 수 있음
- macOS 기본 앱보다 무겁거나 불안정할 수 있음

## 어떤 상황에 어떤 방법이 좋은가

### 개발자/디버깅 목적
QuickTime가 가장 무난하다. 화면 녹화, 캡처, 실기기 화면 확인에 충분하고 macOS 기본 앱이라 관리 포인트가 적다.

### 발표/회의 목적
AirPlay가 편하다. 케이블 없이 바로 화면 공유가 가능하고 발표 흐름이 자연스럽다. 다만 네트워크 상태는 반드시 확인하는 편이 좋다.

### 상시 미러링/복잡한 기능 필요
서드파티 도구가 낫다. 조작, 장시간 녹화, 추가 제어 기능이 필요하면 전용 앱이 더 편할 수 있다.

## 실무에서 자주 하는 선택

내 기준으로는 아래처럼 정리해두면 편하다.

- **Mac 연결 가능 + 안정성 우선** → QuickTime
- **무선 발표 + 회의실 화면 공유** → AirPlay
- **제어/녹화/부가기능 우선** → 서드파티 도구

## 체크리스트

미러링 전에 아래를 보면 실패를 줄일 수 있다.

- 기기 잠금 해제 여부
- 케이블 연결 상태 또는 동일 네트워크 여부
- 신뢰(Trust) 허용 여부
- 회의실 장비가 AirPlay를 지원하는지
- 발표 화면에 알림이 뜨지 않도록 방해 금지 모드가 켜져 있는지

## 마무리

iOS 화면 미러링은 방법이 많지만, 목적에 맞게 고르면 훨씬 간단하다. 개인적으로는 **개발/디버깅은 QuickTime**, **회의실/발표는 AirPlay**가 가장 실용적이었다. 서드파티 프로그램은 기본 기능으로 부족할 때만 추가로 검토하면 된다.

처음부터 모든 도구를 다 깔기보다, 자신의 사용 패턴에 맞는 기본 루트를 하나 정해두는 것이 가장 편하다.
