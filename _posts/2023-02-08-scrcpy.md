# scrcpy란?

`scrcpy`는 Android 기기 화면을 PC에 미러링하고, 키보드/마우스로 제어할 수 있게 해주는 매우 실용적인 도구다. 이름은 "screen copy"에서 왔다.

핵심 특징은 다음과 같다.

- USB 또는 TCP/IP 연결 지원
- 루팅 불필요
- GNU/Linux, Windows, macOS 지원
- 비교적 지연이 적고 가벼움
- 화면 미러링뿐 아니라 제어와 녹화도 가능

Android 디버깅을 자주 한다면 거의 필수 도구에 가깝다.

## 어떤 상황에서 유용한가

- 실기기 앱 데모
- 화면 녹화
- QA 재현 영상 확보
- USB 연결 상태에서 빠른 조작
- 키보드 입력이 많은 테스트
- 모바일 화면을 PC 화면과 같이 보여줘야 할 때

## 1. scrcpy 설치

### macOS

Homebrew가 가장 간단하다.

```bash
brew install scrcpy
```

### Windows

[다운로드 페이지](https://github.com/Genymobile/scrcpy/releases){:target="_blank"}에서 Windows용 zip을 받아 압축 해제한다.

일반적인 흐름:

- Assets에서 `win64-v1.xx.zip` 다운로드
- 예: `C:\Program Files\scrcpy` 에 압축 해제
- 환경변수 `Path`에 해당 경로 추가

![Image Alt scrcpy3](/assets/img/contents/scrcpy/scrcpy3.png)

## 2. 사전 준비사항

scrcpy는 내부적으로 adb 연결을 사용한다. 따라서 아래 두 가지가 먼저 준비되어야 한다.

### PC 쪽
- Android platform-tools 설치
- `adb` 명령 사용 가능 상태
- 관련 글: [adb설치](https://changok89.github.io/utility/adb/)

### Android 폰 쪽
- 개발자 옵션 활성화
- USB 디버깅 ON
- 관련 글: [usb debugging 활성화](https://changok89.github.io/android/AndroidUSBDebugging/)

## 3. USB로 미러링하기

### macOS

1. Mac에 Android 폰을 연결한다.
2. 터미널에서 아래 실행

```bash
scrcpy
```

![Image Alt scrcpy1](/assets/img/contents/scrcpy/scrcpy1.png)

### Windows

1. PC에 Android 폰을 연결한다.
2. 명령 프롬프트에서 실행

```bash
scrcpy
```

![Image Alt scrcpy2](/assets/img/contents/scrcpy/scrcpy2.png)

정상 연결되면 Android 화면이 PC 창으로 뜨고, 마우스와 키보드로 조작할 수 있다.

## 4. 자주 쓰는 옵션

아래 옵션만 알아도 실사용 범위가 꽤 넓다.

| option | description |
|---|---|
| `--always-on-top` | 창을 항상 위에 표시 |
| `-b`, `--bit-rate value` | 비트레이트 지정 |
| `--crop width:height:x:y` | 서버에서 화면 일부만 잘라서 표시 |
| `-f`, `--fullscreen` | 전체화면 시작 |
| `--max-fps value` | FPS 제한 |
| `-m`, `--max-size` | 화면 최대 크기 제한 |
| `-n`, `--no-control` | 장치 제어 비활성화 |
| `-N`, `--no-display` | 화면 표시 없이 실행 |
| `-p`, `--port` | 클라이언트 수신 TCP 포트 설정 |
| `-s`, `--serial` | 여러 기기 중 특정 시리얼 지정 |
| `-S`, `--turn-screen-off` | 기기 화면 즉시 끄기 |
| `-t`, `--show-touches` | 터치 표시 활성화 |

## 5. 화면 녹화

scrcpy는 미러링과 동시에 녹화할 수 있다.

```bash
scrcpy --record=fileName.mp4
```

또는

```bash
scrcpy -r fileName.mkv
```

데모 영상, QA 이슈 재현, 문서용 캡처를 남길 때 편하다.

## 6. 화면 없이 녹화만 하기

PC 화면에 실시간 미러링은 필요 없고 파일만 저장하고 싶다면 아래처럼 사용한다.

```bash
scrcpy --no-display --record=fileName.mp4
```

또는

```bash
scrcpy -Nr fileName.mkv
```

종료는 보통 `Ctrl + C`.

## 7. 무선 ADB 연결

무선으로 쓰려면 먼저 adb 연결을 무선 모드로 바꿔야 한다.

### 순서

1. PC와 Android 폰을 같은 Wi-Fi에 연결한다.
2. 먼저 USB 케이블로 폰을 PC에 연결한다.
3. 아래 명령 실행

```bash
adb tcpip 5555
```

4. USB 케이블을 분리한다.
5. 기기 IP로 연결한다.

```bash
adb connect ${phone-ip}:5555
```

## 8. 무선으로 scrcpy 실행

ADB 무선 연결이 끝났다면 평소처럼 `scrcpy`를 실행하면 된다.

무선 연결은 편하지만 지연이 늘 수 있어서 비트레이트와 해상도를 낮춰 주는 편이 좋다.

```bash
scrcpy --bit-rate=2M --max-size=800
```

또는

```bash
scrcpy -b2M -m800
```

## 9. 잘 안 될 때 체크리스트

### 창이 안 뜬다
- `adb devices`로 기기가 보이는지 먼저 확인
- USB 디버깅이 켜져 있는지 확인
- RSA 허용 팝업을 승인했는지 확인

### 여러 기기가 연결돼 있다
시리얼을 지정한다.

```bash
scrcpy -s [device_serial]
```

### 무선 연결이 느리다
- 비트레이트 낮추기
- 해상도 줄이기
- Wi-Fi 품질 확인

### 제어는 되는데 화면이 불안정하다
- USB 연결 상태 확인
- 고해상도 기기면 `--max-size` 옵션 사용
- 필요하면 FPS 제한도 고려

## 10. scrcpy가 특히 좋은 이유

개인적으로 scrcpy의 가장 큰 장점은 아래 세 가지다.

- 가볍고 빠르다
- 루팅이 필요 없다
- 미러링 + 제어 + 녹화를 한 도구로 해결할 수 있다

그래서 Android 화면을 단순히 보여주는 것을 넘어, **개발/QA 작업 도구**로 쓰기 좋다.

## 마무리

scrcpy는 Android 화면을 PC에서 가장 편하게 다룰 수 있는 도구 중 하나다. 설치도 어렵지 않고, adb만 준비돼 있으면 USB/무선 모두 활용할 수 있다.

빠르게 정리하면:

1. adb 준비
2. USB 디버깅 활성화
3. `scrcpy` 실행
4. 필요하면 녹화, 무선 연결, 비트레이트 조정

실기기 테스트가 많은 환경이라면 한 번 익혀두고 계속 쓰게 되는 도구다.
