# FFmpeg란?

FFmpeg은 오디오와 비디오를 변환, 추출, 병합, 저장할 때 가장 널리 쓰이는 커맨드라인 도구 중 하나다. 포맷 변환뿐 아니라 인코딩, 자막 처리, 스트리밍 소스 저장, 리사이즈, 오디오 처리까지 매우 다양한 작업을 할 수 있다.

실제로는 아래 같은 상황에서 자주 사용한다.

- 동영상 포맷 변환
- HLS(m3u8) 저장
- 자막 포함 영상 생성
- 인코딩 코덱 변경
- 파일 크기 줄이기
- 오디오 추출

## 1. FFmpeg 설치

### Windows

1. [ffmpeg 다운로드](https://www.gyan.dev/ffmpeg/builds/ffmpeg-release-full.7z){:target="_blank"}
2. 압축 해제 후 예를 들어 `C:\Program Files\ffmpeg` 위치로 옮긴다.
3. 시스템 환경변수 `Path`에 `C:\Program Files\ffmpeg\bin` 추가
4. 명령 프롬프트에서 `ffmpeg -version` 실행

### macOS

Homebrew가 가장 편하다.

```bash
brew install ffmpeg
```

설치 후 확인:

```bash
ffmpeg -version
```

## 2. 자주 쓰는 옵션

- `-i` : 입력 파일 지정
- `-vcodec` 또는 `-c:v` : 비디오 코덱 지정
- `-acodec` 또는 `-c:a` : 오디오 코덱 지정
- `-vf` : 비디오 필터 적용 (리사이즈, 자막, 속도 조정 등)
- `-af` : 오디오 필터 적용
- `-b:v` : 비디오 비트레이트 지정
- `-b:a` : 오디오 비트레이트 지정
- `-f` : 출력 포맷 지정
- `-crf` : 품질 기준 압축 수준 조절

처음에는 `-i`, `-c copy`, `-c:v libx264`, `-c:a aac`, `-crf` 정도만 익혀도 실사용에 충분하다.

## 3. 가장 자주 쓰는 패턴

### 컨테이너만 바꾸기 (재인코딩 없이 빠르게)
코덱 호환이 이미 맞으면 `-c copy`가 빠르다.

#### mkv → mp4

```bash
ffmpeg -i source.mkv -c copy dest.mp4
```

#### ts → mp4

```bash
ffmpeg -i source.ts -acodec copy -vcodec copy -f mp4 dest.mp4
```

#### mpg → mp4

```bash
ffmpeg -i source.mpg -acodec copy -vcodec copy -f mp4 dest.mp4
```

이 방식은 빠르지만, 입력 코덱이 출력 컨테이너와 정말 호환되는지 확인해야 한다.

### 재인코딩이 필요한 변환
원본 포맷이나 코덱이 애매하면 `libx264 + aac` 조합이 무난하다.

#### wmv → mp4

```bash
ffmpeg -i source.wmv -c:v libx264 -crf 23 -c:a aac -q:a 100 dest.mp4
```

이 조합은 호환성이 좋고 결과도 비교적 예측 가능하다.

## 4. 포맷별 예시

### avi → mp4

```bash
ffmpeg -i source.avi -bsf:a aac_adtstoasc -c copy dest.mp4
```

### VOB 여러 개 합쳐서 mp4 만들기

```bash
ffmpeg -i "concat:VTS_01.VOB|VTS_02.VOB|VTS_03.VOB" dest.mp4
```

DVD 구조처럼 VOB 파일이 여러 개로 쪼개져 있을 때 유용하다.

## 5. m3u8 저장

### 원격 m3u8 다운로드

```bash
ffmpeg -i https://domain.com/source.m3u8 -bsf:a aac_adtstoasc -vcodec copy -c copy -crf 50 dest.mp4
```

### 로컬 m3u8 저장

```bash
ffmpeg -protocol_whitelist file,http,https,tcp,tls,crypto -i source.m3u8 -c copy -bsf:a aac_adtstoasc dest.mp4
```

HLS 저장은 실제로 자주 쓰지만, DRM이 걸린 스트림은 이 방식으로 해결되지 않는다.

## 6. 언제 `-c copy`를 쓰고, 언제 재인코딩하나

### `-c copy`가 좋은 경우
- 빠르게 컨테이너만 바꾸고 싶을 때
- 원본 품질 손실 없이 저장하고 싶을 때
- 입력 코덱이 출력 포맷과 호환될 때

### 재인코딩이 필요한 경우
- 플레이어 호환성이 중요할 때
- 원본 코덱이 애매할 때
- 용량 최적화가 필요할 때
- 출력 규격을 통일하고 싶을 때

## 7. 자주 만나는 문제

### 변환은 됐는데 재생이 안 된다
컨테이너만 바꿨는데 내부 코덱이 호환되지 않는 경우일 수 있다. 이때는 `libx264 + aac`로 재인코딩을 고려한다.

### 용량이 너무 크다
`-crf` 값을 조절하거나 해상도, 비트레이트를 낮춰야 한다.

### m3u8이 안 저장된다
프로토콜 whitelist, 상대경로 참조, 암호화 여부를 확인해야 한다.

## 8. 실무에서 자주 쓰는 추천 조합

### 안전한 mp4 변환

```bash
ffmpeg -i input.mov -c:v libx264 -crf 23 -c:a aac output.mp4
```

### 빠른 컨테이너 변환

```bash
ffmpeg -i input.mkv -c copy output.mp4
```

### HLS 저장

```bash
ffmpeg -i playlist.m3u8 -c copy -bsf:a aac_adtstoasc output.mp4
```

## 마무리

FFmpeg은 처음엔 옵션이 많아 보여도, 실제로는 몇 가지 패턴만 익히면 대부분의 변환 작업을 해결할 수 있다.

정리하면:

- 빠른 변환 → `-c copy`
- 안전한 재인코딩 → `libx264 + aac`
- 스트리밍 저장 → `m3u8 + protocol/bitstream 옵션`

영상 변환 작업이 반복된다면 자신이 자주 쓰는 명령 템플릿을 몇 개 정리해두는 것이 가장 효율적이다.
