# ADB(Android Debug Bridge) 란?

`adb`는 Android 기기와 PC가 통신할 수 있도록 도와주는 대표적인 명령줄 도구다. Android 개발, QA, 디버깅, 자동화 작업에서 거의 기본 도구처럼 쓰인다.

대표적으로 아래 작업을 할 수 있다.

- 연결된 기기 확인
- 쉘 진입
- APK 설치 / 제거
- 파일 push / pull
- 로그 확인
- 무선 디버깅 연결

또한 `adb` 클라이언트는 기본적으로 5037 포트를 사용해 adb 서버와 통신한다. 자세한 구조 설명은 공식 문서를 참고하면 된다.  
[Android adb 공식 문서](https://developer.android.com/studio/command-line/adb?hl=ko){:target="_blank"}

## 1. ADB 설치

ADB는 Android SDK 전체를 설치하지 않아도 `platform-tools`만 받아서 사용할 수 있다.

### 설치 순서

1. [platform-tools 다운로드](https://developer.android.com/studio/releases/platform-tools?hl=ko){:target="_blank"}
2. 운영체제에 맞는 압축 파일을 내려받아 적절한 위치에 압축 해제
3. 터미널 또는 명령 프롬프트에서 `adb`를 실행할 수 있도록 PATH 설정

Android SDK를 설치했다면 보통 아래와 같은 위치에 `platform-tools`가 있다.

```bash
/Users/${사용자}/Library/Android/sdk/platform-tools
```

### macOS 예시
`.zshrc` 등에 PATH를 추가한다.

![Image Alt paths](/assets/img/contents/adb/paths.png)

설치 후에는 아래 명령으로 확인한다.

```bash
adb version
adb devices
```

## 2. 사용 전 준비

ADB가 설치돼 있어도 기기에서 디버깅이 열려 있지 않으면 실제로 아무것도 못 한다. 아래를 먼저 확인해야 한다.

- Android 기기에서 개발자 옵션 활성화
- USB 디버깅 ON
- 처음 연결 시 RSA 허용 팝업 승인
- 데이터 통신 가능한 USB 케이블 사용

## 3. 자주 쓰는 ADB 명령어

### `adb devices`
연결된 장치 목록을 확인한다.

```bash
adb devices
```

![Image Alt adb_devices](/assets/img/contents/adb/adb_devices.png)

가장 먼저 실행하는 명령이다. 여기서 기기가 안 보이면 그 뒤 단계는 거의 의미가 없다.

### `adb shell`
기기 셸로 들어간다.

```bash
adb shell
```

![Image Alt adb_shell](/assets/img/contents/adb/adb_shell.png)

파일 탐색, 패키지 정보 확인, 시스템 명령 실행 등에 사용한다.

### `adb start-server`
ADB 서버를 시작한다.

```bash
adb start-server
```

![Image Alt adb_start_server](/assets/img/contents/adb/adb_start_server.png)

### `adb kill-server`
ADB 서버를 종료한다.

```bash
adb kill-server
```

기기 인식이 꼬였을 때 `kill-server` 후 `start-server`로 다시 정리하는 경우가 많다.

### `adb pull`
기기에서 PC로 파일을 가져온다.

```bash
adb pull [source] [dest]
```

![Image Alt adb_pull](/assets/img/contents/adb/adb_pull.png)

로그 파일, 다운로드 파일, 앱이 생성한 결과물을 가져올 때 자주 쓴다.

### `adb push`
PC에서 기기로 파일을 복사한다.

```bash
adb push [source] [dest]
```

![Image Alt adb_push](/assets/img/contents/adb/adb_push.png)

테스트용 파일을 기기에 넣거나 앱이 읽을 데이터를 미리 복사할 때 편하다.

### APK 설치

```bash
adb install -r file.apk
```

`-r`은 기존 앱이 있으면 교체 설치한다.

### 특정 기기에 APK 설치
기기가 여러 대 연결돼 있으면 시리얼을 지정해야 한다.

```bash
adb -s [device_serial] install -r file.apk
```

### APK 제거

```bash
adb uninstall package.name
```

패키지명으로 제거한다. 파일명으로 지우는 게 아니라는 점을 주의한다.

## 4. 무선 연결

USB 없이 ADB를 쓰고 싶다면 TCP/IP 모드로 전환해 무선 연결할 수 있다.

### 1단계: 포트 열기

```bash
adb -s [device_serial] tcpip 5555
```

![Image Alt adb_network_setting](/assets/img/contents/adb/adb_network_setting.png)

### 2단계: 기기 IP로 연결

```bash
adb connect [ip_address]:5555
```

![Image Alt adb_network_connect](/assets/img/contents/adb/adb_network_connect.png)

무선 연결은 케이블을 자주 뺐다 꽂아야 하는 환경에서 특히 편하다.

## 5. 어떤 상황에서 ADB를 자주 쓰나

- APK 설치/삭제
- WebView 디버깅 전 연결 확인
- scrcpy 실행 전 연결 확인
- 앱 내부 파일 가져오기
- 로그 및 패키지 상태 점검
- 테스트 자동화 스크립트 실행

즉 Android 실기기 작업의 출발점이라고 봐도 된다.

## 6. 잘 안 될 때 체크리스트

### `adb devices`에 아무 것도 안 뜸
- USB 디버깅 켜져 있는지 확인
- 기기 잠금 해제 여부 확인
- 데이터 케이블인지 확인
- 드라이버 문제(Windows) 확인

### `unauthorized` 로 표시됨
- 기기에서 RSA 허용 팝업을 승인
- 필요하면 `adb kill-server && adb start-server` 후 재시도

### 무선 연결이 안 됨
- 같은 네트워크인지 확인
- 기기 IP가 바뀌지 않았는지 확인
- 방화벽 또는 보안 정책 확인

## 7. 같이 알아두면 좋은 명령

자주 쓰는 최소 세트만 기억해도 생산성이 많이 올라간다.

- `adb devices`
- `adb shell`
- `adb install -r app.apk`
- `adb pull`
- `adb push`
- `adb kill-server`
- `adb connect ip:port`

## 마무리

ADB는 Android 개발/디버깅에서 빠질 수 없는 기본 도구다. 설치 자체는 단순하지만, 실제로는 USB 디버깅, RSA 허용, 케이블 상태, 무선 연결 방식까지 함께 알아야 제대로 쓸 수 있다.

처음에는 `devices`, `shell`, `install`, `push/pull` 정도만 익혀도 대부분의 실기기 작업이 훨씬 쉬워진다.
